本文档旨在为开发者提供openJiuwen框架的系统要求、权限声明、安装步骤及使用规范，确保框架在集成和运行过程中的稳定性、安全性。

# **系统要求**

## **操作系统支持**

- **Windows**：最低版本Windows 10，推荐版本Windows 11。
- **macOS**：最低版本10.14，推荐版本12.0及以上。
- **Linux**：
  - 支持Ubuntu 18.04 LTS及以上，推荐Ubuntu 20.04 LTS、Ubuntu 22.04 LTS；
  - 支持Debian 10及以上，推荐Debian 11、Debian 12；
  - 支持RHEL 7及以上，推荐RHEL 9；
  - 支持Fedora 34及以上，推荐Fedora 38、Fedora 39。

## **环境依赖要求**

- ​本框架的运行环境的Python版本应高于或者等于Python 3.11版本，建议使用3.11.4版本，使用前请检查Python版本信息。

# **磁盘空间要求**

## **安装空间**

- ​**最小安装空间**​：约 ​**200MB**​（含依赖项）。

## **日志存储空间**

- openJiuwen框架本身不强制写日志，但如果出于商用目的，建议预留 **1GB\~5GB** 用于本地调试日志。

# **权限要求**

## **日志路径权限**

- 用户需确保日志路径具备**读写权限**，否则openJiuwen框架将无法正常记录日志，可能导致调试困难或服务异常。
- 若日志路径为系统路径（如 `/var/log/`），需确保运行账户对该路径有**写入权限**。
- **若日志路径为 root 权限路径（如 `/var/log/xxx-sdk/`），用户需提供 root 权限或配置 sudo 权限**，否则openJiuwen框架将无法正常写入日志。

## **网络权限**

- 需允许openJiuwen框架访问外部网络，用于调用外部依赖的三方服务（例如：大模型服务、插件服务）。

# **配置要求**

## 通过配置文件进行配置

在执行流程前，开发者可以通过调用配置解析类`UserConfig`的`set_config_path`方法，使得自定义配置生效。（入参为`config.ini`配置文件的路径，路径不得为敏感路径，例如`/etc/passwd`、`/etc/shadow`、`/etc/hosts`、`/etc/hostname`、`C:\Windows\System32\`、`C:\Windows\SysWOW64\`、`C:\Windows\System\`）。

示例代码如下：

```python
UserConfig.set_config_path(Path("./config.ini"))
```

### **隐私敏感信息的日志打印配置**

若需打印包含敏感信息（包括但不限于：用户输入的请求、用户与智能体的对话历史、大模型输入输出）的日志，则需要在配置文件中进行相应的配置。若`is_sensitive`配置为`true`，表示不打印隐私敏感数据；若配置为`false`，表示打印隐私敏感数据，默认值为`true`。

```ini
[settings]
is_sensitive = false
```

### 非安全路径黑名单配置

通过配置文件可以指定非安全路径黑名单。一旦配置了某个路径为非安全路径，则openJiuwen框架将不支持读取该路径下的任意文件。

开发者自定义的非安全路径黑名单，多个路径用英文逗号间隔；若不配置，框架使用默认黑名单（具体包括：`/etc/passwd`、`/etc/shadow`、`/etc/hosts`、`/etc/hostname`、`/etc/ssh/`、`C:\Windows\System32\`、`C:\Windows\SysWOW64\`、`C:\Windows\System\`）。

```ini
[settings]
sensitive_paths = /etc/passwd,/etc/shadow
```

## 通过环境变量进行配置

通过设置环境变量，可以进行如下配置。

### SSL校验开关配置

openJiuwen为了实现默认安全，默认打开SSL校验开关：对外发送请求时，默认开启HTTPS SSL证书校验，以保证数据链路的安全。具体针对调用大模型服务和调用三方插件服务，相关的环境变量配置如下：

> **注意**
> 本地测试请求服务（http 开头），可以通过配置相关的环境变量关闭SSL校验。但禁用SSL会跳过证书验证，可能遭遇数据篡改、中间人攻击，导致敏感信息泄露，仅允许测试环境临时使用，生产环境务必启用SSL校验保障安全。


- **调用大模型服务：**
  
  - **LLM_SSL_VERIFY**：调用大模型服务的SSL校验开关。默认配置为`true`，表示开启HTTPS SSL证书校验；如果开发者不需要开启HTTPS SSL证书校验，则需要手动配置为`false`。
  - **LLM_SSL_CERT**：调用大模型服务的证书路径。当`LLM_SSL_VERIFY=true`，需要配置本地的证书路径；当`LLM_SSL_VERIFY=false`，则无需配置证书路径。其默认配置为`""`，表示使用系统默认根证书库。
- **调用三方服务：**
  
  - **RESTFUL_SSL_VERIFY**：调用三方插件服务的SSL校验开关。默认配置为`true`，表示开启HTTPS SSL证书校验；如果开发者不需要开启HTTPS SSL证书校验，则需要手动配置为`false`。
  - **RESTFUL_SSL_CERT**：调用三方插件服务的证书路径。当`RESTFUL_SSL_VERIFY=true`，需要配置本地的证书路径；当`RESTFUL_SSL_VERIFY=false`，则无需配置证书路径。其默认配置为`""`，表示使用系统默认根证书库。

### SSRF防护开关配置

openJiuwen为了保障默认安全，默认打开SSRF（Server-Side Request Forgery，服务器端请求伪造）防护开关：通过禁止敏感地址（如本地IP、内网地址），攻击者利用服务器端程序的 “请求转发” 功能，注入自定义目标地址。

> **注意**
> 开发者在本地测试环境，如需请求敏感地址（如本地IP、内网地址），必须将`SSRF_PROTECT_ENABLED`环境变量手动设置为`false`。但是禁用SSRF防护可能会导致攻击者诱导服务器向指定地址发送未授权请求，窃取内部资源或攻击内网的漏洞，因此生产环境务必启用SSRF防护。

- **SSRF_PROTECT_ENABLED**：开启SSRF防护的开关。默认配置为`true`，表示开启SSRF防护。当前认定内网地址均属于敏感地址，一旦识别到内网地址，openJiuwen框架将抛出异常。
内网地址的认定范围包括：`10.0.0.0`-`10.255.255.255`、`172.16.0.0`-`172.31.255.255`、`192.168.0.0`-`192.168.255.255`、`127.0.0.0`-`127.255.255.255`、`0.0.0.0`。
如果开发者在本地测试环境不需要开启SSRF开启，则需要手动配置为`false`。
