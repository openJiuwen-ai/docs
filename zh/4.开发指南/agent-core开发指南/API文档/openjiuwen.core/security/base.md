# openjiuwen.core.security

当前版本中，`openjiuwen.core.security` 包仅作为**安全相关能力的扩展入口**，源码 `__init__.py` 中的 `__all__` 为空，没有对外导出的类或函数。

这意味着：

- 目前业务开发者**不需要**也**不应该**直接依赖本包中的任何 API；
- 当未来在该包中增加具体的安全能力（例如鉴权、审计、敏感信息脱敏等组件）时，会在此处补充相应的类和方法文档。

> 之所以在 API 文档中保留 `openjiuwen.core.security`，主要是为了保证文档目录结构与源码结构一一对应，方便后续扩展和维护。
