# 接入大模型

不同模型在推理能力、对话流畅性、多轮交互等方面各有特色，用户可根据具体的应用场景灵活选择最适合的模型，例如在需要复杂逻辑推理的场景下选择推理能力更强的模型，在追求自然对话体验的场景下选择交互更自然的模型。

openJiuwen通过 `openjiuwen.core.foundation.llm` 提供统一的**模型客户端 + 配置**体系，推荐的接入方式是：

- 使用 `ModelClientConfig` 描述「如何连接模型服务」（client_provider/api_base/api_key/SSL 等）；
- 使用 `ModelRequestConfig` 描述「这次调用用哪个模型 + 调用参数」（model/temperature/top_p 等）；
- 使用统一入口类 `Model`，通过 `invoke/stream` 进行调用。框架会根据 `client_provider` 自动选择对应的模型客户端实现。

## 使用 Model 调用模型

### 初始化模型

以硅基流动厂商的模型接入为例：

```python
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
)

# 1. 配置客户端连接信息（client_provider/api_base/api_key 等）
model_client_config = ModelClientConfig(
    client_provider="SiliconFlow",              # 模型提供商标识符，框架根据此值自动选择客户端实现
    api_base="https://api.siliconflow.cn/v1",   # 模型服务 URL
    api_key="sk-****************************",  # 鉴权 Token
    verify_ssl=False                            # 示例中关闭 SSL 校验，生产环境建议开启
)

# 2. 配置本次请求的模型及参数
model_request_config = ModelRequestConfig(
    model="Qwen/Qwen3-32B",     # 具体模型名称，由服务端支持的模型列表决定
    temperature=0.7,
    top_p=0.9,
)

# 3. 创建统一模型入口
model = Model(
    model_client_config=model_client_config,
    model_config=model_request_config,
)
```

> **说明**
> - 用户需要自行前往硅基流动或 OpenAI 等厂商官网注册账号，以获取可用模型的 `api_key` 和模型调用的 URL 地址 `api_base`。
> - `client_provider` 目前内置支持 `OpenAI` 和 `SiliconFlow`，框架会根据此配置自动选择对应的模型客户端实现。

### 准备模型输入

大模型支持以下主要输入参数，仅配置 `messages` 时，模型将基于上下文直接生成回复；同时配置 `messages` 和 `tools` 时，模型将结合上下文信息与工具定义，判断是否调用工具以完成用户请求。

- **messages**：按对话顺序排列的消息列表，每个消息对象包含发送者的角色（`role`）和消息内容（`content`），为**必选**参数。入参支持两种常见写法（`List[BaseMessage]`、`List[Dict]`）：

**写法一：使用 BaseMessage 类型**

```python
from openjiuwen.core.foundation.llm import SystemMessage, UserMessage

messages = [
    SystemMessage(content="你是一个AI助手"),
    UserMessage(content="你好")
]
```

**写法二：使用字典类型**

```python
messages = [
    {"role": "system", "content": "你是一个AI助手"},
    {"role": "user", "content": "你好"}
]
```

- **tools**：大模型可使用的工具列表，每个工具使用 JSON Schema 格式定义，详细说明所需参数、参数类型及是否必填等，为**可选**参数，仅在需要执行工具调用时使用。示例代码如下：

```python
# 工具定义
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get current weather for a location",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {
                    "type": "string",
                    "description": "City and country e.g. Paris, France"
                },
                "units": {
                    "type": "string",
                    "enum": ["metric", "imperial"],
                    "description": "Temperature unit"
                }
            },
            "required": ["location"]
        }
    }
}]
```

### 调用模型

大模型的调用支持非流式 `invoke` 和流式 `stream` 两种方式，均为**异步方法**：

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| `invoke` | 异步非流式调用，一次性返回完整结果 | 需要一次性拿到完整结果、对实时性要求不高的场景 |
| `stream` | 异步流式调用，逐块返回响应内容 | 需要高并发实时推送、异步非阻塞输出，或边生成边展示的场景 |

#### async invoke 方法

异步调用大模型，一次性获取完整响应结果。

**参数**：

- **messages**(Union[str, List[BaseMessage], List[dict]])：调用大模型的输入消息。必填。
- **tools**(Union[List[ToolInfo], List[dict], None], 可选)：指定模型可调用的工具列表。默认值：None。
- **temperature**(float, 可选)：控制模型输出的随机性。取值范围：[0, 1]。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **top_p**(float, 可选)：控制模型输出的多样性。取值范围：[0, 1]。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **model**(str, 可选)：指定模型名称。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **max_tokens**(int, 可选)：生成的最大 token 数量。默认值：None。
- **stop**(str, 可选)：停止序列。默认值：None。
- **output_parser**(BaseOutputParser, 可选)：输出解析器，用于解析模型响应内容。默认值：None。
- **timeout**(float, 可选)：本次请求的超时时间。覆盖 ModelClientConfig 中的配置。默认值：None（使用配置值）。

**返回**：

**AssistantMessage**，大模型返回的消息对象，包含以下主要字段：
- `role`：角色，固定为 `assistant`。
- `content`：响应内容文本。
- `tool_calls`：工具调用列表（如果有）。
- `usage_metadata`：使用统计信息。
- `finish_reason`：完成原因，`stop` 表示正常结束，`tool_calls` 表示需要调用工具。

**示例**：

```python
import asyncio
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
    SystemMessage,
    UserMessage,
)


async def invoke_example():
    # 配置模型客户端
    model_client_config = ModelClientConfig(
        client_provider="SiliconFlow",
        api_base="https://api.siliconflow.cn/v1",
        api_key="sk-****************************",
        verify_ssl=False,
    )
    model_request_config = ModelRequestConfig(
        model="Qwen/Qwen3-32B",
        temperature=0.7,
        top_p=0.95,
    )
    model = Model(
        model_client_config=model_client_config,
        model_config=model_request_config,
    )

    # 准备模型输入数据
    messages = [
        SystemMessage(content="你是一个AI助手"),
        UserMessage(content="杭州天气")
    ]

    # 天气工具 schema 定义
    tools = [{
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City and country e.g. Paris, France"
                    },
                    "units": {
                        "type": "string",
                        "enum": ["metric", "imperial"],
                        "description": "Temperature unit"
                    }
                },
                "required": ["location"]
            }
        }
    }]

    # 调用模型
    response = await model.invoke(messages=messages, tools=tools)
    print(response)


if __name__ == "__main__":
    asyncio.run(invoke_example())
```

**输出结果**：

```
AssistantMessage(role='assistant', content='', tool_calls=[ToolCall(id='019afe192ceee268d980f8acd98ccbc1', type='function', name='get_weather', arguments='{"location": "杭州, 中国"}')], usage_metadata=UsageMetadata(model_name='Qwen/Qwen3-32B', input_tokens=156, output_tokens=23, total_tokens=179), finish_reason='tool_calls')
```

#### async stream 方法

异步流式调用大模型，逐块返回响应内容。

**参数**：

参数与 `invoke` 方法相同。

**返回**：

**AsyncIterator[AssistantMessageChunk]**，异步迭代器，每次迭代返回一个响应块，包含以下主要字段：
- `content`：当前块的内容增量。
- `reasoning_content`：推理内容（如果模型支持）。
- `tool_calls`：工具调用增量（如果有）。
- `usage_metadata`：使用统计信息（通常只在最后一个块中包含）。
- `finish_reason`：完成原因。

**示例**：

```python
import asyncio
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
    SystemMessage,
    UserMessage,
)


async def stream_example():
    # 配置模型客户端
    model_client_config = ModelClientConfig(
        client_provider="SiliconFlow",
        api_base="https://api.siliconflow.cn/v1",
        api_key="sk-****************************",
        verify_ssl=False,
    )
    model_request_config = ModelRequestConfig(
        model="Qwen/Qwen3-32B",
        temperature=0.7,
        top_p=0.95,
    )
    model = Model(
        model_client_config=model_client_config,
        model_config=model_request_config,
    )

    # 准备模型输入数据
    messages = [ 
        SystemMessage(content="你是一个AI助手"),
        UserMessage(content="你好")
    ]

    # 使用 async for 遍历异步迭代器
    async for chunk in model.stream(messages=messages):
        print(chunk.content, end="", flush=True)
    print()  # 换行


if __name__ == "__main__":
    asyncio.run(stream_example())
```

**输出结果**：

```
你好
！有什么我可以
帮你的吗
？
```
