执行期能力由 **`Session` + `Runner` + `ResourceManager`** 等模块共同承担：

- **Session（会话）**：状态管理、交互、中断恢复、流式输出与 trace 等统一入口（`openjiuwen.core.session`）。
- **Runner（执行器）**：负责驱动 Agent/Workflow 执行与资源管理（`openjiuwen.core.runner`）。
- **Workflow（工作流）**：以 `Workflow.invoke(inputs, session, context=...)` 为主要执行入口（`openjiuwen.core.workflow`）。

执行期体系的核心优势包括：

* **有状态会话管理**：维护任务执行过程中的完整会话与中间状态，支持事务性操作与断点续传，确保复杂任务的一致性与可靠性。
* **动态化配置管理**：通过 `Session.get_workflow_config()` / `Session.get_agent_config()` 等接口在执行期读取配置。
* **结构化消息流**：原生支持流式文本与结构化中间结果（如推理步骤、工具调用事件）的实时推送，为前端提供沉浸式交互体验。
* **受控执行流程**：提供标准化的中断、暂停、回滚与继续执行机制，赋予开发者和系统对长周期任务前所未有的精细控制力。
* **深度可观测性**：自动追踪并记录从推理链、工具调用到API请求的全链路遥测数据，为性能剖析、成本审计与根因分析提供唯一可信源。
* **统一资源治理**：通过 Runner/资源管理器统一管理大模型、工具函数、提示模板、工作流等，实施生命周期与配额控制。

## 基本功能

* **配置管理**：通过 `Session` 读取 Agent/Workflow 配置。
* **状态管理**：通过 `Session` 的 `get_state/get_global_state` 与 `update_state/update_global_state` 进行读写。
* **资源管理**：Runner 及资源管理器统一管理工作流、提示词、工具及大模型等重量级实例。
* **调测能力**：通过 `Tracer`/stream schema 等实现全链路观测与调试。
* **流式输出**：通过 `Session.write_stream` / `Session.stream_iterator` 等能力实现结构化流式输出。
* **中断恢复**：通过 Checkpointer 保存快照并恢复执行，结合交互输出实现无缝续跑。
