工作流与 Agent 的配置信息由会话 `Session`/`Config` 承载，并在执行期间通过 `Session` 提供统一读取入口，实现配置信息的管理。

- **工作流配置**：工作流的 meta 配置（例如 id/name/version/description 等）。通过 `session.get_workflow_config(workflow_id=...)` 获取。
- **Agent配置**：Agent 的配置（例如 id/version/description 等）。通过 `session.get_agent_config()` 获取。

下面以工作流组件为例，演示如何在组件执行期间通过 `session` 获取工作流配置与 Agent 配置。

# 获取工作流配置信息

使用 `session.get_workflow_config` 方法可获取指定工作流 id 的工作流配置信息：

```python
# 注意：此代码片段依赖于会话/资源初始化，不能单独运行。
workflow_config = session.get_workflow_config(workflow_id="test_workflow")
print(workflow_config.metadata.id)
print(workflow_config.metadata.name)
print(workflow_config.metadata.version)
```

**参数说明**：

- **workflow_id**：要读取配置的工作流 id。
  - 该 id 通常来自工作流 card（例如 `WorkflowCard.id`），用于唯一标识某个工作流。
  - 不能为空；若传 `None`，底层会抛出异常（用于避免“读取不到配置但静默返回 None”导致难排查问题）。
  - 建议与资源管理器注册时使用的 workflow_id 完全一致，否则会读到 `None` 或触发异常（取决于实现）。

获取到的id、name和version信息如下：

```python
test_weather_agent
weather
1.0
```

完整代码参考[完整用例](配置管理.md#完整用例)章节。

# 获取Agent配置信息

使用 `session.get_agent_config` 方法可获取 Agent 的配置信息：

```python
# 注意：此代码片段依赖于会话/资源初始化，不能单独运行。
agent_config = session.get_agent_config()
print(agent_config.id)
print(agent_config.version)
```

获取到的id和version信息如下:

```python
test_weather_agent
0.1.0
```

完整代码参考[完整用例](配置管理.md#完整用例)章节。

# 组件中读取配置（示例）

```python
from openjiuwen.core.workflow import WorkflowComponent
from openjiuwen.core.context_engine import ModelContext
from openjiuwen.core.session.node import Session
from openjiuwen.core.graph.executable import Input, Output


class CustomComponent(WorkflowComponent):
    def __init__(self, workflow_id: str):
        super().__init__()
        self._workflow_id = workflow_id

    async def invoke(self, inputs: Input, session: Session, context: ModelContext) -> Output:
        workflow_config = session.get_workflow_config(self._workflow_id)
        agent_config = session.get_agent_config()

        return {
            "workflow_id": workflow_config.metadata.id,
            "workflow_version": workflow_config.metadata.version,
            "agent_id": agent_config.id,
            "agent_version": agent_config.version,
        }
```

> **提示**：工作流/Agent 配置的注册与加载由 Runner 与资源管理器负责。
>
> 组件侧只通过 `session` 读取：这样组件不会和具体加载方式耦合（例如本地文件/远端注册表/动态下发）。

# 完整用例

以上各节示例代码的组合即为完整用例：先通过 `session.get_workflow_config(workflow_id)` 获取工作流配置，再通过 `session.get_agent_config()` 获取 Agent 配置，或在自定义组件中按需读取。完整可运行示例请参考仓库中的 Session/配置管理相关示例代码。
