# 流式输出

流式输出能力由 `Session` 承载（见 `openjiuwen.core.session`）。组件在执行期间可以通过会话写入标准流式帧或自定义流式帧，外部可通过工作流/Agent 的 stream 相关接口消费。

## 核心数据结构：OutputSchema

`OutputSchema` 是推荐的标准化流式帧结构（见 `openjiuwen.core.session.stream`）。通常包含：

- **type**：帧类型。
  - 建议用稳定的枚举/字符串（例如 `answer`、`step`、`tool_call`、`interaction`）。
  - 上层 UI/消费端通常据此决定渲染方式。
- **index**：帧序号。
  - 用于保证同一条流式输出的顺序一致（尤其在并发/重试场景下）。
  - 建议从 0 开始递增，或使用能够反映阶段的序号策略。
- **payload**：帧载荷。
  - 建议是可 JSON 序列化的 dict（避免写入不可序列化对象）。
  - 建议包含足够上下文（例如内容、阶段、耗时、引用 id 等），以便上层复现与调试。

## 工作流流式输出

工作流组件可以在 `invoke/stream/transform` 等执行过程中，通过 `session.write_stream(...)` 写入标准化流式数据。

```python
from openjiuwen.core.workflow import WorkflowComponent
from openjiuwen.core.context_engine import ModelContext
from openjiuwen.core.session.node import Session
from openjiuwen.core.graph.executable import Input, Output
from openjiuwen.core.session.stream import OutputSchema


class StreamOutComp(WorkflowComponent):
    async def invoke(self, inputs: Input, session: Session, context: ModelContext) -> Output:
        await session.write_stream(OutputSchema(type="answer", index=0, payload={"content": "hello"}))
        return {"answer": "hello"}
```

### 参数说明：write_stream

```python
await session.write_stream(data)
```

- **data**：要写入的流式帧。
  - 推荐传 `OutputSchema`，确保上层解析稳定。
  - 若传 dict，则由会话实现决定如何封装（不建议在团队内混用多种封装格式）。

## 自定义流式输出（custom stream）

当你需要输出“业务自定义结构”且不希望被标准 schema 约束时，可使用 `session.write_custom_stream(...)`。

典型场景：前端需要渲染业务卡片/表单/图表等结构化内容。

```python
await session.write_custom_stream({
    "kind": "metric",
    "name": "retrieval_hit_rate",
    "value": 0.82,
    "detail": {
        "top_k": 5,
        "source": "vector_store"
    }
})
```

## Agent 流式输出

Agent 会话侧同样支持 `write_stream` / `stream_iterator`（见 `openjiuwen.core.session.agent.Session`），用于把 Agent 执行过程中的中间帧实时输出到上层消费端。

> 说明：流式数据结构（如 `OutputSchema`）以源码与 `openjiuwen.core.session.stream` 模块定义为准。
>
> 生产场景建议统一帧类型与 payload 结构，避免上层解析出现分支爆炸。
