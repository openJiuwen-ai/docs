# 资源管理

资源管理能力主要由 Runner 与资源管理器承担（见 `openjiuwen.core.runner`），统一管理并复用以下资源：

- **Workflow**：工作流实例与注册表
- **ModelClient/Model**：大模型客户端与模型实例
- **PromptTemplate/Instruction**：提示词模板与指令模板
- **Tool**：本地工具、RESTful API 工具、MCP 工具等

组件与工作流执行期间如果需要读取相关配置，一般通过 `Session` 获取（例如 `session.get_workflow_config(...)` / `session.get_agent_config()`）。

## 会话侧获取资源（Task/Agent 场景）

在 Agent/任务执行场景下，框架会提供带资源能力的会话（例如 `openjiuwen.core.session.internal.wrapper.TaskSession`）。它会把资源管理器注入到底层会话，从而允许在执行期间按 id 获取资源实例：

- `get_model(model_id)`
- `get_tool(tool_id)`
- `get_prompt(template_id)`
- `get_workflow(workflow_id)` / `get_workflow_sync(workflow_id)`

**参数语义（通用）**：

- **model_id/tool_id/template_id/workflow_id**：资源唯一标识符。
  - 一般与资源注册表中的 id 保持一致。
  - 建议在团队内形成“id 命名规范”（例如 `weather_openai_gpt4o`、`tool.search_web`、`prompt.weather.system_v1`），避免冲突。
  - id 不存在时通常会抛异常或返回 `None`（取决于具体资源类型与实现），建议在业务侧做好兜底与错误提示。

**示例（示意）**：

```python
# 说明：示例展示“会话侧取资源”的基本方式，具体初始化/注册由 Runner 负责。

model = session.get_model(model_id="gpt-4o-mini")
tool = session.get_tool(tool_id="search_web")
prompt = session.get_prompt(template_id="weather.system.v1")

# workflow 获取一般是异步（可能涉及加载/编译）
flow = await session.get_workflow(workflow_id="weather_flow")
```

> 具体 API 与使用方式以源码 `openjiuwen.core.runner.resources_manager` 与 `openjiuwen.core.session.internal.wrapper.TaskSession` 为准。
