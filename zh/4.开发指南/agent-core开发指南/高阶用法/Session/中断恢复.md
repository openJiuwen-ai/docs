# 中断恢复

“中断恢复”由 `Session` 的交互（interaction）与 checkpoint（checkpointer）能力共同实现：

- **交互中断**：组件通过 `session.interact(...)` 发起交互，工作流可进入 `INPUT_REQUIRED` 状态，等待用户继续输入。
- **异常中断与恢复**：通过 checkpointer 保存会话与工作流状态，重新执行时从上次中断位置继续（具体恢复策略以 Runner/Workflow 实现为准）。

## 交互输入/输出模型

交互体系通常包含两类结构：

- **InteractionOutput**：执行侧“向外发起交互”的输出结构（包含 id 与提示 value）。
  - id 通常是节点/组件 id，用于让上层知道“是谁在等待输入”。
  - value 通常是提示文本或结构化表单描述。
- **InteractiveInput**：恢复侧“把用户输入送回去”的输入结构。
  - 可用 `raw_inputs` 表示首轮输入，也可用 `update(node_id, value)` 绑定到特定节点 id。
  - 这两种写法不要混用：一旦 `raw_inputs` 已存在，就不能再按 node_id update（避免语义冲突）。

## 交互中断（示例）

```python
from openjiuwen.core.workflow import WorkflowComponent
from openjiuwen.core.context_engine import ModelContext
from openjiuwen.core.session.node import Session
from openjiuwen.core.graph.executable import Input, Output


class InteractiveComp(WorkflowComponent):
    async def invoke(self, inputs: Input, session: Session, context: ModelContext) -> Output:
        cmd = inputs.get("cmd")
        confirm = await session.interact(f"Do you want to execute '{cmd}'?")
        return {"confirm_result": confirm}
```

### 参数说明：interact

```python
confirm = await session.interact(value)
```

- **value**：交互提示内容。
  - 推荐传字符串，确保上层 UI/调用方可以直接展示。
  - 也可以传结构化 dict（例如需要用户填写的字段、默认值、校验规则），由上层按约定渲染。
  - 建议包含足够上下文（例如要确认的命令/关键参数），避免用户无法理解自己在确认什么。

### 恢复执行（示意）

当执行进入 `INPUT_REQUIRED`/中断态后，上层收集到用户输入即可构造 `InteractiveInput` 并继续调用工作流（具体入口以 Runner/Workflow 实现为准）。示意如下：

```python
from openjiuwen.core.session.interaction import InteractiveInput
from openjiuwen.core.session.workflow import create_workflow_session

# 复用同一个 session_id（确保能命中此前保存的 checkpoint）
session = create_workflow_session(session_id="your_session_id")

# 将用户输入绑定到交互 id（通常为节点 id）
inputs = InteractiveInput()
inputs.update(node_id="interactive_node_id", value="yes")

# 继续执行（此处为示意：具体参数/入口以实际 workflow.invoke 约定为准）
await flow.invoke({"user_inputs": inputs}, session)
```

## 相关 API

- `openjiuwen.core.session.interaction`：`InteractiveInput` / `InteractionOutput` / `AgentInterrupt`
- `openjiuwen.core.session.checkpointer`：`Checkpointer` / `InMemoryCheckpointer`
