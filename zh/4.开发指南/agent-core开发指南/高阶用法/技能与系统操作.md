# 技能与系统操作（Skills）

Skills（技能）和系统操作（SysOperation）是 openJiuwen 在当前版本中新增的一组能力，用于把**磁盘上的技能知识 + 受控系统能力**安全地开放给 Agent 使用。

本章节基于源码 `openjiuwen.core.skills` 与 `openjiuwen.core.sys_operation`，介绍如何：

- 通过 `SkillUtil` 为 Agent 注册技能目录；
- 自动为 Agent 挂载查看文件、执行代码、执行命令等内置工具；
- 将技能信息注入到 Agent 的提示词中。

> 说明：下面示例只使用已经在源码中真实存在的类/方法（例如 `SkillUtil.register_skills`、`SkillUtil.get_skill_prompt`），不再使用旧版已删除的接口。

---

## 1. 能力总览

### 1.1 Skills 模块的角色

源码入口：`openjiuwen.core.skills`，对外主要导出三个类：

- `SkillManager`：维护技能注册表，负责扫描技能目录、解析 `Skill.md` 元数据。
- `SkillToolKit`：基于 SysOperation 创建内置工具（查看文件、执行 Python、执行 Shell）。
- `SkillUtil`：面向业务的一站式封装，组合前两者完成“注册技能 + 挂载工具 + 生成提示词”。

在高阶用法中，大部分场景只需要直接使用 `SkillUtil` 即可。

### 1.2 SysOperation 的作用

源码入口：`openjiuwen.core.sys_operation`，对外导出：

- `OperationMode`：运行模式枚举（如 local / sandbox）。
- `LocalWorkConfig`、`SandboxGatewayConfig`：本地 / 沙箱模式下的配置。
- `SysOperationCard`：描述一个 SysOperation 实例的卡片。
- `SysOperation`：实际对接文件系统、代码执行、Shell 的入口类。

Skills 模块不会直接访问操作系统，而是全部通过 SysOperation 的三类子接口完成：

- `sys_op.fs()`：读写文件、列目录等；
- `sys_op.code()`：执行代码，如 Python 代码块；
- `sys_op.shell()`：执行 Shell 命令。

> 在实际工程项目中，SysOperation 实例通常由 Runner 的资源管理器在启动时统一注册，并分配一个约定好的 `sys_operation_id`（例如 `"default_sys_op"`）。Skills 相关代码只需要拿到这个 id 即可，无需关心具体注册细节。

## 2. 准备技能目录（Skill.md）

Skills 通过扫描磁盘目录来发现技能。一个典型的技能目录结构如下：

```text
skills_root/
  ├─ data_analysis/
  │    ├─ Skill.md
  │    └─ examples/ ...
  └─ sql_knowledge/
       ├─ Skill.md
       └─ cheatsheet.sql
```

每个子目录中必须包含一个 `Skill.md` 文件。

在源码中，`SkillManager` 只依赖 `Skill.md` 的 YAML front matter 中的 `description` 字段，用于生成技能描述，其余内容由业务自由定义，例如：

```markdown
---
name: sql_knowledge
description: 为 Agent 提供常用 SQL 语法、优化技巧等知识
---

这里是技能的详细说明，可以包含示例、最佳实践等。
```

要点：

- 技能名称默认取自目录名（如 `sql_knowledge`）。
- `description` 会被展示在技能提示词中，建议写成简明但信息量足够的中文说明。

## 3. 使用 SkillUtil 为 Agent 挂载技能

### 3.1 创建 SkillUtil

`SkillUtil` 是高阶用法的主要入口，典型初始化方式如下：

```python
from openjiuwen.core.skills import SkillUtil


async def setup_agent_with_skills(agent, sys_operation_id: str = "default_sys_op"):
    # 1. 创建 SkillUtil，并指定要使用的 SysOperation id
    skill_util = SkillUtil(sys_operation_id=sys_operation_id)

    # 2. 注册技能目录并为 Agent 挂载技能相关工具
    await skill_util.register_skills(
        skill_path="/path/to/skills_root",  # skills_root 目录路径
        agent=agent,
        session_id="session_001",
    )

    # 3. 生成技能提示词片段，可拼接到 Agent 的系统提示词中
    skill_prompt = skill_util.get_skill_prompt()
    print(skill_prompt)
```

上述代码中用到的所有接口都可以在源码中找到：

- `SkillUtil.__init__(sys_operation_id: str)`
- `SkillUtil.register_skills(skill_path: str, agent, session_id: str | None = None) -> bool`
- `SkillUtil.get_skill_prompt() -> str`

SkillUtil 在内部会做两件事：

- 调用 `SkillToolKit.add_skill_tools(agent)`，为 Agent 挂载内置工具；
- 调用 `SkillManager.register(Path(skill_path), session_id)`，扫描技能目录并注册技能。

### 3.2 内置工具一览

根据 `openjiuwen.core.skills.skill_tool_kit.SkillToolKit` 的实现，当前会为 Agent 挂载三个本地工具（均基于 `LocalFunction` 实现）：

- `view_file`：查看文本文件内容。
  - 对应内部工具 id：`"_internal_view_file"`。
  - 主要入参：`file_path: str`。
  - 内部通过 `sys_operation.fs().read_file(file_path, mode="text")` 读取文本内容。
- `execute_python_code`：执行一段 Python 代码。
  - 对应内部工具 id：`"_internal_execute_python_code"`。
  - 主要入参：`code_block: str`。
  - 内部通过 `sys_operation.code().execute_code(code_block, language="python")` 执行，并返回 stdout / stderr。
- `run_command`：执行 Shell 命令。
  - 对应内部工具 id：`"_internal_run_command"`。
  - 主要入参：`bash_command: str`。
  - 内部通过 `sys_operation.shell().execute_cmd(bash_command)` 执行命令。

> 这些工具会自动注册到 Runner 的资源管理器中，同时其 `ToolCard` 会被挂到 Agent 的 `ability_kit` 上。如何在 ReAct Agent 中调用 Tool，请参考《构建ReActAgent.md》中关于 Tool 使用的说明。

## 4. 在 ReAct / Workflow Agent 中结合 Skills 使用

Skills 本身只负责“提供工具 + 提供技能提示词”，与具体 Agent 类型解耦。一个常见模式是：

1. 定义一个基于 ReAct 的 Agent（例如《构建ReActAgent.md》中介绍的 `LLMAgent` 或其他单 Agent）。
2. 在 Agent 初始化完成后，调用前文的 `setup_agent_with_skills(agent, sys_operation_id)` 为其挂载技能。
3. 在构造 Agent 的系统提示词时，将 `skill_util.get_skill_prompt()` 拼接进去，引导大模型按需使用技能和内置工具。

伪代码示例（省略 Agent 构造细节，仅展示拼接思路）：

```python
BASE_SYSTEM_PROMPT = "你是一个能够阅读项目代码、执行命令的智能体。"

async def build_agent_with_skills():
    agent = create_my_react_agent()  # 这里复用构建ReActAgent中的示例

    skill_util = SkillUtil(sys_operation_id="default_sys_op")
    await skill_util.register_skills(
        skill_path="/path/to/skills_root",
        agent=agent,
        session_id="session_001",
    )

    skill_prompt = skill_util.get_skill_prompt()
    agent.system_prompt = BASE_SYSTEM_PROMPT + "\n\n" + skill_prompt

    return agent
```

> 由于 `SkillUtil` 与 Agent 类型之间只通过 `agent.ability_kit` 和 ToolCard 进行交互，因此上述用法同样适用于 WorkflowAgent 或其他自定义 Agent。

## 5. 与记忆、检索等模块的协同

在一个完整的智能体应用中，Skills 往往会与记忆引擎、知识检索等模块协同工作：

- 通过记忆引擎（`LongTermMemory`）管理用户画像和变量；
- 通过知识检索（`SimpleKnowledgeBase` 等）管理外部文档知识；
- 通过 Skills 扩展“操作能力”，如：
  - 查看项目中的配置文件或代码片段；
  - 在安全环境中执行少量调试代码；
  - 执行受控的 Shell 命令（例如 `ls` 或 `cat`）。

这些模块在源码中彼此独立、接口清晰，可以按需组合。使用时只需确保：

- 文档中涉及的类与方法（如 `SkillUtil.register_skills`、`SkillUtil.get_skill_prompt`、`OperationMode` 等）在源码中真实存在；
- 实际工程中按照业务需求合理配置 SysOperation 的运行模式与工作目录，避免越权访问。
