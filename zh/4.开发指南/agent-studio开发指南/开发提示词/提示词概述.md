# 概述

提示词管理是一套完整的提示词生命周期管理解决方案，帮助您高效创建、编辑、调试、优化和管理提示词。

## 核心功能

### 1. 灵活的消息模板构建

支持四种消息类型，构建完整的对话上下文：

- **System消息**：定义AI角色、行为准则、输出格式。
- **User消息**：模拟用户输入，提供任务上下文。
- **Assistant消息**：细化角色指导，或提供历史对话上下文。
- **Placeholder消息**：为动态内容预留位置，运行时注入真实上下文。

### 2. 双模板引擎模式

- **Normal模式**：简单变量替换，使用 `{{变量名}}` 语法，自动识别变量，适合80%的常规场景。
- **Jinja2模式**：专业模板引擎，支持条件判断、循环遍历等复杂逻辑，适合20%的复杂场景。

### 3. 多实例运行与对比模式

- **多实例运行**：同时运行多个实例，批量测试并横向对比输出质量，验证提示词稳定性。
- **对比模式**：创建基准组和对照组，在同一输入下对比不同提示词或配置的效果，快速找到最佳方案。

### 4. 智能提示词优化

提供多种优化方式，帮助您持续提升提示词质量：

- **快捷优化**：一键对整个提示词进行全局优化，快速查看优化前后对比。
- **反馈优化**：根据具体的优化需求，对提示词进行精准的局部或全局优化。支持三种精准优化模式，根据具体需求改进提示词。
  - 全文反馈优化：对整个提示词进行全局优化。
  - 选中反馈优化：对选中的文本片段进行优化。
  - 插入反馈优化：在光标位置插入优化后的内容。
- **基于调试结果优化**：基于实际调试对话中的AI回复，通过人工评估来优化提示词。
- **用例集驱动的自优化**：通过用例集自动迭代优化提示词，系统会根据用例集的实际输出结果与预期输出进行对比，自动优化提示词以达到目标准确率。

## 核心概念

- **提示词（Prompt）**：指导AI模型生成特定输出的文本指令，由多个消息组成完整的对话上下文。
- **消息类型**：System（角色定义）、User（用户输入）、Assistant（细化指导/历史上下文）、Placeholder（动态内容）。
- **变量（Variable）**：提示词中的占位符，用于在运行时动态替换内容，通过 `{{变量名}}` 语法引用。
- **模板引擎**：Normal模式（简单变量替换）和Jinja2模式（复杂逻辑处理）。
- **自优化**：基于用例集自动进行多轮优化，持续提升提示词准确率。
