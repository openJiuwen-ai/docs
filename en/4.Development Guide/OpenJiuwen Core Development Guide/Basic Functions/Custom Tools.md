openJiuwen supports the following two ways to convert developer-defined tools into tools that can be recognized and invoked by LLMs:

- Toolifying local functions: Provides the `@tool` decorator, which can register local Python functions as a Tool class recognizable by LLMs, enabling Function Calling-like capabilities.
- Wrapping Restful interfaces: Provides the `RestfulApi` class to conveniently define and manage Restful-style API interfaces, including interface name, description, parameters, path, headers, request method, and response information, enabling unified and standardized interface management and invocation.

# Toolifying Local Functions

openJiuwen provides the `@tool` decorator to wrap local Python functions into a `Tool` class, making them recognizable and callable by LLMs.

## Using the @tool decorator to wrap local functions

Developer-defined local Python functions can be wrapped using the `@tool` decorator. The `@tool` decorator requires the developer to configure the function name, description, and parameters to accurately describe the local Python function; it is recommended to refer to the function’s definition when configuring.

```python
from openjiuwen.core.utils.tool.param import Param
from openjiuwen.core.utils.tool.tool import tool

# Developer implements the add function and wraps it with the tool decorator
@tool(name="add",
    description="Local addition plugin",
    params=[
        Param(name="a", description="First parameter", type="int", required=True),
        Param(name="b", description="Second parameter", type="int", required=True),
    ])
def add(a: int, b: int) -> int:
    return a + b
```

## Invoking the tool

The `Tool` class generated by the `@tool` decorator can be recognized and invoked by LLMs. Prepare the corresponding tool inputs according to the parameter definitions in the `@tool` decorator, then call the Tool class’s `invoke` method to trigger the execution logic of the local function.

```python
# Generate Tool class, recognizable and callable by LLMs
tool_info = add.get_tool_info()
print(f"Invocation succeeded, returned result: {tool_info}")

inputs = {
    "a": 5,
    "b": 1
}
result = add.invoke(inputs=inputs)
print(f"Invocation succeeded, returned result: {result}")
```

Sample output:

```text
Invocation succeeded, returned result: type='function' name='add' description='Local addition plugin' parameters=Parameters(type='object', properties={'a': {'description': 'First parameter', 'type': 'string'}, 'b': {'description': 'Second parameter', 'type': 'string'}}, required=['a', 'b'])
Invocation succeeded, returned result: 6
```

# Encapsulating Restful APIs

openJiuwen provides the `RestfulApi` class to wrap Restful API services; the resulting `RestfulApi` class can be recognized and invoked by LLMs.

## Using the RestfulApi class to wrap Restful API services

After developers implement and deploy a Restful API service, they can use the `RestfulApi` class provided by openJiuwen for wrapping. The `RestfulApi` class requires configuration of the interface name, description, parameters, path, headers, request method, and response information to describe the Restful API service. It is recommended to refer to the Restful API service’s interface when configuring.

```python
import asyncio

from openjiuwen.core.utils.tool.service_api.restful_api import RestfulApi
from openjiuwen.core.utils.tool.param import Param

# Developers implement and deploy a weather Restful API service, then create a RestfulApi instance for the weather query plugin
weather_plugin = RestfulApi(
    name="WeatherReporter",
    description="Weather query plugin",
    params=[
        Param(name="location", description="Location for the weather query, must be in English", type="string", required=True),
        Param(name="date", description="Date for the weather query, in format YYYY-MM-DD", type="string", required=True),
    ],
    path="your weather search api url", # Deployment address of the weather query service
    headers={},
    method="GET",
    response=[],
)
```

## Invoking the tool

The `RestfulApi` class can be recognized and invoked by LLMs. Refer to the parameter definitions of the `RestfulApi` class, prepare the tool inputs, and calling the `RestfulApi` class’s `invoke` method will run the Restful API service remotely.

```python
# RestfulApi instance, recognizable and callable by LLMs
tool_info = weather_plugin.get_tool_info()
print(f"Invocation succeeded, returned result: {tool_info}")

inputs = {
    "location": "beijing",
    "date": "2023-10-01"
}
loop = asyncio.get_event_loop()
result = loop.run_until_complete(weather_plugin.ainvoke(inputs=inputs))
print(f"Invocation succeeded, returned result: {result}")
```

Sample output:

```text
Invocation succeeded, returned result: type='function' name='WeatherReporter' description='Weather query plugin' parameters=Parameters(type='object', properties={'location': {'description': 'Location for the weather query, must be in English', 'type': 'string'}, 'date': {'description': 'Date for the weather query, in format YYYY-MM-DD', 'type': 'string'}}, required=['location', 'date'])
Invocation succeeded, returned result: The weather in beijing on 2023-10-01 is sunny
```
