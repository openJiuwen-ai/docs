# openjiuwen.core.utils.tool.schema

## class openjiuwen.core.utils.tool.schema.Parameters

```python
openjiuwen.core.utils.tool.schema.Parameters(type: str = "object", properties: Dict[str, Any] = {}, required: List[str] = [])
```

Defines the structured schema for tool parameter descriptions.

**Parameters**:

- **type** (str): Parameter type identifier, fixed as "object" (indicating a structured object). Default: "object".
- **properties** (Dict[str, Any]): A collection of tool input parameters where keys are parameter names and values are detailed definitions (e.g., type, description). Default: empty dictionary.
- **required** (List[str]): A list of required parameter names indicating which parameters are mandatory when invoking the tool. Default: empty list, indicating no required parameters.

**Examples**:

```python
>>> params = Parameters(
...     properties={
...         "city": {"type": "string", "description": "City name"},
...         "units": {"type": "string", "description": "Temperature unit: 'c' for Celsius, 'f' for Fahrenheit"}
...     },
...     required=["city"]
... )
```

### class openjiuwen.core.utils.tool.schema.ToolInfo

```python
ToolInfo(type: str = Field(default="function"), name: str = Field(default=""), description: str = Field(default=""), parameters: Optional[Parameters] = Field(default=None))
```

Describes the basic information of a tool, including its type, name, description, and parameter schema.

**Parameters**:

- **type** (str): Tool type identifier, fixed as "function". Default: "function".
- **name** (str): The tool/function name, which must uniquely identify the tool. Default: empty string.
- **description** (str): The tool/function description explaining its purpose. Default: empty string.
- **parameters** (Parameters, optional): The structured parameter definition as a Parameters instance. Use None when there are no parameters. Default: None.

**Examples**:

```python
>>> params = Parameters(
...     properties={"city": {"type": "string", "description": "City name"}},
...     required=["city"]
... )
>>> tool_info = ToolInfo(
...     name="get_weather",
...     description="Query real-time weather by city name",
...     parameters=params
... )
```

### class openjiuwen.core.utils.tool.schema.ToolCall

```python
ToolCall(id: Optional[str], type: str, name: str, arguments: str, index: Optional[int] = None)
```

Records a single tool call generated by a model, including the call ID, tool type, name, and raw arguments.

**Parameters**:

- **id** (str, optional): Unique identifier for the tool call, used to trace a single invocation. None if not provided. No default.
- **type** (str): The type of tool being called, must match the ToolInfo type (typically "function"). No default.
- **name** (str): The name of the tool/function being called, must match the ToolInfo name. No default.
- **arguments** (str): The raw argument string for the tool call (typically JSON). No default.
- **index** (int, optional): The position index of the tool call in a sequence, used for ordering multiple tool calls. Default: None.

**Examples**:

```python
>>> tool_call = ToolCall(
...     id="call_890123",
...     type="function",
...     name="get_weather",
...     arguments='{"city":"Beijing","units":"c"}'
... )
>>> # Example without call ID
>>> tool_call_no_id = ToolCall(
...     type="function",
...     name="get_weather",
...     arguments='{"city":"Guangzhou"}'
... )
```
