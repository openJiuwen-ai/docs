openJiuwen supports the following three ways to convert developer-defined tools into tools that can be recognized and invoked by LLMs:

- Toolifying local functions: Provides the `@tool` decorator, which can register local Python functions as a Tool class recognizable by LLMs, enabling Function Calling-like capabilities.
- Wrapping Restful interfaces: Provides the `RestfulApi` class to conveniently define and manage Restful-style API interfaces, including interface name, description, parameters, path, headers, request method, and response information, enabling unified and standardized interface management and invocation.
- Wrapping MCP interfaces: Provides the `MCPTool` class to conveniently wrap MCP services, enabling unified interface management and invocation of MCP services.

# Toolifying Local Functions

openJiuwen provides the `@tool` decorator to wrap local Python functions into a `Tool` class, making them recognizable and callable by LLMs.

## Using the @tool decorator to wrap local functions

Developer-defined local Python functions can be wrapped using the `@tool` decorator. The `@tool` decorator requires the developer to configure the function name, description, and parameters to accurately describe the local Python function. It is recommended to refer to the local function's definition when configuring.

```python
from openjiuwen.core.foundation.tool import tool, Input, Output
from openjiuwen.core.foundation.tool.schema import ToolInfo

# Developer implements the add function and wraps it with the tool decorator
@tool(name="add",
    description="Local addition plugin",
    params=[
        Param(name="a", description="First parameter", type="int", required=True),
        Param(name="b", description="Second parameter", type="int", required=True),
    ])
def add(a: int, b: int) -> int:
    return a + b
```

## Invoking the tool

The `Tool` class generated by the `@tool` decorator can be recognized and invoked by LLMs. Prepare the corresponding tool inputs according to the parameter definitions in the `@tool` decorator, then call the Tool class's `invoke` method to trigger the execution logic of the local function.

```python
# Generate Tool class, recognizable and callable by LLMs
tool_info = add.get_tool_info()
print(f"Invocation succeeded, returned result: {tool_info}")

inputs = {
    "a": 5,
    "b": 1
}
result = add.invoke(inputs=inputs)
print(f"Invocation succeeded, returned result: {result}")
```

Sample output:

```text
Invocation succeeded, returned result: type='function' name='add' description='Local addition plugin' parameters=Parameters(type='object', properties={'a': {'description': 'First parameter', 'type': 'string'}, 'b': {'description': 'Second parameter', 'type': 'string'}}, required=['a', 'b'])
Invocation succeeded, returned result: 6
```

# Encapsulating Restful APIs

openJiuwen provides the `RestfulApi` class to wrap Restful API services. The resulting `RestfulApi` class can be recognized and invoked by LLMs.

## Using the RestfulApi class to wrap Restful API services

After developers implement and deploy a Restful API service, they can use the `RestfulApi` class provided by openJiuwen for wrapping. The `RestfulApi` class requires configuration of the interface name, description, parameters, path, headers, request method, and response information to describe the Restful API service. It is recommended to refer to the Restful API service's interface when configuring.

```python
import asyncio

from openjiuwen.core.foundation.tool.service_api.restful_api import RestfulApi, RestfulApiCard

# Developers implement and deploy a weather Restful API service, then create a RestfulApi instance for the weather query plugin
weather_plugin = RestfulApi(
    name="WeatherReporter",
    description="Weather query plugin",
    params=[
        {"name": "location", "description": "Location for the weather query, must be in English", "type": "string", "required": True},
        {"name": "date", "description": "Date for the weather query, in format YYYY-MM-DD", "type": "string", "required": True},
    ],
    path="your weather search api url", # Deployment address of the weather query service
    headers={},
    method="GET",
    response=[],
)
```

## Invoking the tool

The `RestfulApi` class can be recognized and invoked by LLMs. Refer to the parameter definitions of the `RestfulApi` class, prepare the tool inputs, and calling the `RestfulApi` class's `ainvoke` method will run the Restful API service remotely.

```python
# RestfulApi instance, recognizable and callable by LLMs
tool_info = weather_plugin.get_tool_info()
print(f"Invocation succeeded, returned result: {tool_info}")

inputs = {
    "location": "beijing",
    "date": "2023-10-01"
}
loop = asyncio.get_event_loop()
result = loop.run_until_complete(weather_plugin.ainvoke(inputs=inputs))
print(f"Invocation succeeded, returned result: {result}")
```

Sample output:

```text
Invocation succeeded, returned result: type='function' name='WeatherReporter' description='Weather query plugin' parameters=Parameters(type='object', properties={'location': {'description': 'Location for the weather query, must be in English', 'type': 'string'}, 'date': {'description': 'Date for the weather query, in format YYYY-MM-DD', 'type': 'string'}}, required=['location', 'date'])
Invocation succeeded, returned result: The weather in beijing on 2023-10-01 is sunny
```

# Encapsulating MCP Interfaces

openJiuwen provides the `MCPTool` class to wrap MCP services. The resulting `MCPTool` class can be recognized and invoked by LLMs.

## Using the MCPTool class to wrap MCP services

After developers implement and deploy an MCP service, they can use the `MCPTool` class provided by openJiuwen for wrapping. The `MCPTool` class requires configuration of information such as MCP Client, input Schema, and service name to describe the MCP service. Currently, multiple Transport methods are supported, including SSE, stdio, and playwright.

```python
import asyncio
from openjiuwen.core.foundation.tool.mcp.base import MCPTool, SseClient

# Developers implement and deploy a weather MCP service, then create a MCPTool instance for the weather query plugin
mcp_client = SseClient(server_path="your weather mcp url", name="MockSseClient")
await mcp_client.connect(timeout=10)
tool_info_list = await mcp_client.list_tools()
for tool_info in tool_info_list:
    print("Invocation succeeded, returned result:", tool_info.model_dump_json())
mcp_tool = MCPTool(mcp_client=mcp_client, tool_info=tool_info_list[0])
```

Sample output:

```text
Invocation succeeded, returned result: {"type":"function","name":"query_weather","description":"","parameters":null,"input_schema":{"properties":{"location":{"title":"Location","type":"string"}},"required":["location"],"title":"query_weatherArguments","type":"object"},"server_name":""}
```

## Invoking the tool

The `MCPTool` class can be recognized and invoked by LLMs. Refer to the parameter definitions of the `MCPTool` class, prepare the tool inputs, and calling the `MCPTool` class's `ainvoke` method will run the MCP service remotely.

```python
# MCPTool instance, recognizable and callable by LLMs
tool_info = mcp_tool.get_tool_info()
print(f"Invocation succeeded, returned result: {tool_info}")

inputs = {
    "location": "beijing",
}
result = await mcp_tool.ainvoke(inputs=inputs)
print(f"Invocation succeeded, returned result: {result}")
await mcp_client.disconnect()
```

Sample output:

```text
Invocation succeeded, returned result: type='function' name='query_weather' description='' parameters=None input_schema={'properties': {'location': {'title': 'Location', 'type': 'string'}}, 'required': ['location'], 'title': 'query_weatherArguments', 'type': 'object'} server_name=''
Invocation succeeded, returned result: {'result': '{\n  "location": "beijing",\n  "temperature": "22℃",\n  "condition": "sunny"\n}'}
```
